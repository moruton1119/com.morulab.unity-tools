# Booth Library Manager Asset Manager

**機能概要**
外部アプリケーション「BOOTH Library Manager (BLM)」のデータを直接読み込み、Unity内でショップのようにアセットを管理・インポートできるツールです。

**主な機能**

- **BLM連携**: BLMのデータベース (`data.db`) から購入済みアセットリストを取得。
- **サムネイル表示**: アセットのサムネイルをタイル状に表示。
- **パッケージ選択**: 1つの商品フォルダ内に複数の `.unitypackage` がある場合、個別に選択してインポート可能。
- **インポートキュー**: 大量のパッケージを順次インポートすることで、Unityのクラッシュを防止。
- **ローカルモード**: BLMに登録されていない自作アセットなども同様のUIで管理可能。
- **カスタムリスト**: BLMアプリで作成したリストでフィルタリング可能。
- **インポート管理**: インポート先フォルダの表示・削除が可能。

**使い方**

## 基本操作

1. **Window > Morulab > BLM Connector** を開きます。
2. リストからインポートしたいアセットをクリックします。
3. 右側に表示される詳細パネルで、インポートしたいパッケージにチェックを入れます。
4. **Add to Import Queue** を押すと、順次インポートが開始されます。

---

## フィルター機能

サイドバーのドロップダウンから、表示するアセットをフィルタリングできます。

| 選択肢 | 内容 |
|--------|------|
| All Products | BLM + LocalAssets の全アセットを表示 |
| BLM Products | BLMデータベースのアセットのみ表示 |
| Local Products | LocalAssetsフォルダのアセットのみ表示 |
| 📋 [リスト名] | BLMで作成したカスタムリスト内のアセットのみ表示 |

### カスタムリストについて

BOOTH Library Managerアプリで作成したリストが自動的に読み込まれます。
リストの作成・編集はBLMアプリ側で行ってください。

## インポート方法

### 方法1: ダブルクリック（クイックインポート）
製品カードをダブルクリックすると、その製品の全パッケージを一括インポートできます。

### 方法2: 個別インポート
1. 製品をシングルクリックしてモーダルを開く
2. 各ファイルの「Import」ボタンをクリック
3. 必要なファイルだけインポート可能

### 方法3: キューシステム
1. 複数の製品を「Add to Queue」でキューに追加
2. 「Process Queue」で一括インポート
3. 「Show Import Dialog」のON/OFFでダイアログ表示を制御

---

## インポート済みアセットの管理

インポート済みの製品には、モーダルフッターに2つのボタンが表示されます。

### Show in Project
- インポート先のフォルダをProjectウィンドウでフォーカス・ハイライトします
- どのフォルダにインポートされたかすぐに確認できます

### Delete
- インポート先のフォルダを削除します
- 確認ダイアログが表示されます
- BLM管理ラベル（BLM_PID_xxx）が付いているフォルダのみ対象

---

## インポート状態の表示

グリッド上の製品カードで、インポート状態を確認できます。

| 状態 | 表示 | 説明 |
|------|------|------|
| 未インポート | 通常の明るさ | まだUnityにインポートしていない |
| インストール済み | 薄く表示（半透明） | 過去にインポート済み |
| バッチインポート済み | 青枠 | 今回のセッションでインポート完了 |

---

## LocalAssets機能

BLMで管理されていないアセットも、LocalAssetsフォルダに入れることで同じUIで管理できます。

### 使い方
1. サイドバーの「Open LocalAssets Folder」をクリック
2. LocalAssetsフォルダ内にアセット用のフォルダを作成
3. フォルダ名が製品名として表示されます
4. サムネイル（thumbnail.png）を置くと表示されます

---

**SQLiteの導入について**
本ツールの動作には `Mono.Data.Sqlite.dll` もしくは `Microsoft.Data.Sqlite` が必要です。
プロジェクト内に存在しない場合、エラーが表示されます。

**設定について**
DBの参照パスを変更したい場合は、`BLMDatabaseService.cs` 内のパス指定を編集してください。
デフォルトでは BLM の標準インストールパスを参照しています。

---

## トラブルシューティング

| 問題 | 原因 | 解決策 |
|------|------|--------|
| アセットが表示されない | DBが読み込めていない | Refresh DBボタンをクリック |
| Show in Projectボタンが出ない | インポート済みでない | まずインポートしてください |
| フィルターが効かない | リストが読み込めていない | Refresh DBボタンをクリック |
| ラベルが付かない | 古いバージョン | パッケージを最新版に更新 |
